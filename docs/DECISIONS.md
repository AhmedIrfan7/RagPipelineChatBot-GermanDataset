# Decision Log — Phase 0 (Scope)

Records confirmed scope vs open assumptions. Assumptions marked ⚠️ need client
sign-off before the affected step ships. Confirmed items ✅ are locked.

Last updated: build step 1.

---

## 1. Channels
- ✅ **Confirmed:** launch on an **embeddable web chat widget**. WhatsApp / Telegram /
  email are deferred; can be added later via an n8n glue layer (see PLAN → n8n verdict).

## 2. Languages
- ✅ **Confirmed:** German (primary) + English (secondary). Reply language mirrors
  the user's query language. No other languages in scope.

## 3. Human handoff target
- ✅ **Confirmed:** on low confidence / out-of-scope, hand off via **email**. The
  destination address is client-provided and stored in the `HANDOFF_EMAIL`
  environment variable (gitignored `.env`) — never hardcoded in the public repo.

## 4. Pricing document delivery
- ✅ **Confirmed:** pricing answers return **both** an itemized price summary **and**
  a downloadable link to the official PDF price sheet for that class/variant.

## 5. Authoritative price generation
- ✅ **Confirmed from data:** `2025_01_*` sheets are the current generation.
  `2024_06_*` sheets are superseded duplicates (Class C/CE/BA family) and must be
  archived, never served. Every served price states its effective date.
- **Needs client:** confirm no price sheet newer than `2025_01` exists.

## 6. Definition of "wrong answer" (all are release-blocking failures)
- ✅ Wrong **price** (any euro amount not matching the current verified sheet).
- ✅ Wrong **class/variant** (answering for B when user meant BE, B197, etc.).
- ✅ **Outdated** price (serving a superseded sheet).
- ✅ **Fabricated** price/discount/bundle the sheets do not contain.
- ✅ Answering a **pricing** question while the class is still ambiguous (must ask first).

## 7. Success metric (accuracy contract)
- ✅ **100% price accuracy** on a golden test set covering every class-variant,
  enforced as an automated regression gate on every change (PLAN steps 81–82).
- ✅ On anything below the retrieval/grounding confidence threshold, the bot
  **refuses and hands off** rather than guessing (PLAN steps 67–71).

---

## Standing engineering rules
- Prices are **never** generated by the LLM — served verbatim from the structured
  store and validated against it before send.
- Confidential client content (PDFs + any extracted prices/business info) stays in
  the gitignored `data/` tree; only code/schemas/docs are public.
