# RagPipelineChatBot-GermanDataset

Production-grade bilingual (DE/EN) chatbot for a German driving school
(**Fahrschule**). It answers customer questions about driving-license classes,
prices, and enrollment with an **accuracy-first architecture**: prices are never
generated by the language model — they are served verbatim from a deterministic,
human-verified price store. Descriptive/policy questions use retrieval (RAG)
with strict grounding and confidence gating.

> **Core principle:** the chatbot must **never** give a wrong price. Numbers come
> from a structured source of truth; the LLM only routes, clarifies, and phrases.

---

## Why this design

Driving-license pricing is exact and safety-critical for the client. Naive RAG
(vector search + free-form LLM generation) can hallucinate numbers or retrieve
the wrong license class. This project therefore splits the system:

| Concern | Handled by |
| --- | --- |
| Exact prices | **Structured price store** (SQLite/JSON), human-verified, versioned |
| Class disambiguation (e.g. Class B has ~15 variants) | Curated decision tree from the client's own consultation script |
| Descriptive/policy text (hours, prerequisites) | **RAG** with metadata filtering + confidence threshold |
| Intent, follow-up questions, DE/EN phrasing | **LLM orchestration** (LangGraph) — never invents numbers |
| "Never wrong" guarantees | Numeric grounding checks, source binding, refusal-on-low-confidence, human handoff |

## Repository layout

```
.
├── data/            # LOCAL ONLY — confidential client PDFs & derived data (gitignored)
│   ├── raw/         # source PDFs (place client dataset here)
│   ├── interim/     # intermediate extraction artifacts
│   └── processed/   # verified structured price store + RAG chunks
├── src/             # pipeline & application source code
├── tests/           # unit / regression / accuracy (golden) tests
├── docs/            # PLAN.md (88-step build plan) and design docs
├── scripts/         # one-off operational scripts (extraction, verification)
└── requirements.txt # Python dependencies (added during setup steps)
```

## Confidentiality

The `data/` tree holds a real client's confidential documents (price sheets,
business questionnaire, consultation protocols). It is **gitignored in full** and
must never be committed to this public repository. To run the pipeline locally,
place the client PDFs in `data/raw/`.
